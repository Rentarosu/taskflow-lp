APIドキュメント

TaskFlow REST APIを使って、ワークフローを自動化し外部サービスと連携しましょう。

はじめに

TaskFlow APIは、RESTful設計に基づいたHTTP APIです。すべてのリクエストとレスポンスはJSON形式で処理されます。APIを利用することで、タスクの作成・更新、プロジェクトの管理、ユーザー情報の取得など、TaskFlowの主要機能をプログラムから操作できます。

ベースURL

https://api.taskflow.jp/v1

すべてのAPIエンドポイントはHTTPSを必須とし、HTTP経由のリクエストは拒否されます。レスポンスにはUTF-8エンコーディングが使用されます。

認証

TaskFlow APIはAPIキーによる認証を採用しています。APIキーはダッシュボードの「設定 → API」から発行できます。リクエスト時にはAuthorizationヘッダーにBearerトークンとして含めてください。

HTTP Header 
Authorization: Bearer YOUR_API_KEY

APIキーはワークスペース単位で発行されます。キーを第三者と共有しないよう、安全に管理してください。万が一漏洩した場合は、ダッシュボードから即座に無効化できます。

エンドポイント一覧

タスク

GET /tasks タスク一覧を取得
POST /tasks 新規タスクを作成
PUT /tasks/{id} タスクを更新
DELETE /tasks/{id} タスクを削除

プロジェクト

GET /projects プロジェクト一覧を取得
POST /projects 新規プロジェクトを作成
PUT /projects/{id} プロジェクト情報を更新

ユーザー

GET /users ワークスペースメンバー一覧
GET /users/me 自分のユーザー情報を取得

コメント

GET /tasks/{id}/comments タスクのコメント一覧
POST /tasks/{id}/comments コメントを追加
DELETE /comments/{id} コメントを削除

リクエスト例

以下は、curlを使ってタスク一覧を取得する例です。

curl 
# タスク一覧を取得
curl -X GET https://api.taskflow.jp/v1/tasks \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json"

# 新規タスクを作成
curl -X POST https://api.taskflow.jp/v1/tasks \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "title": "デザインレビューを完了する",
    "project_id": "proj_abc123",
    "assignee_id": "user_xyz789",
    "priority": "high",
    "due_date": "2026-03-20"
  }'

レート制限

APIリクエストにはプランに応じたレート制限が適用されます。制限を超えるとHTTP 429ステータスが返されます。

プラン リクエスト上限 期間
Free 100リクエスト 1時間あたり
Pro 1,000リクエスト 1時間あたり
Business 10,000リクエスト 1時間あたり

レスポンスヘッダーの X-RateLimit-Remaining で残りリクエスト数を確認できます。
